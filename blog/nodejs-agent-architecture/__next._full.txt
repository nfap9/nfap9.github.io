<h2>一、Agent 的本质：一个编排循环</h2>

<pre><code>接收输入 → 注入上下文 → 调用 LLM → 解析响应 → 执行工具 → 把结果喂回去 → 再次调用 LLM

<span class="line"><span style="color:#F97583">    for</span><span style="color:#F97583"> await</span><span style="color:#E1E4E8"> (</span><span style="color:#F97583">const</span><span style="color:#79B8FF"> chunk</span><span style="color:#F97583"> of</span><span style="color:#E1E4E8"> stream) {</span></span>
<span class="line"><span style="color:#F97583">      if</span><span style="color:#E1E4E8"> (chunk.type </span><span style="color:#F97583">===</span><span style="color:#9ECBFF"> "content"</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#E1E4E8">        content </span><span style="color:#F97583">+=</span><span style="color:#E1E4E8"> chunk.delta;</span></span>
<span class="line"><span style="color:#F97583">        yield</span><span style="color:#E1E4E8"> chunk; </span><span style="color:#6A737D">// 流式吐给外层</span></span>
<span class="line"><span style="color:#E1E4E8">      } </span><span style="color:#F97583">else</span><span style="color:#F97583"> if</span><span style="color:#E1E4E8"> (chunk.type </span><span style="color:#F97583">===</span><span style="color:#9ECBFF"> "tool_call"</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#E1E4E8">        hasToolCall </span><span style="color:#F97583">=</span><span style="color:#79B8FF"> true</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#6A737D">        // 收集工具调用参数</span></span>
<span class="line"><span style="color:#E1E4E8">      }</span></span>
<span class="line"><span style="color:#E1E4E8">    }</span></span>
<span class="line"></span>
<span class="line"><span style="color:#6A737D">    // 4. 把 assistant 的回复加入对话历史</span></span>
<span class="line"><span style="color:#79B8FF">    this</span><span style="color:#E1E4E8">.thread.</span><span style="color:#B392F0">addMessages</span><span style="color:#E1E4E8">([assistantMessage]);</span></span>
<span class="line"></span>
<span class="line"><span style="color:#F97583">    if</span><span style="color:#E1E4E8"> (</span><span style="color:#F97583">!</span><span style="color:#E1E4E8">hasToolCall) {</span></span>
<span class="line"><span style="color:#E1E4E8">      continueLoop </span><span style="color:#F97583">=</span><span style="color:#79B8FF"> false</span><span style="color:#E1E4E8">; </span><span style="color:#6A737D">// 没有工具调用，任务结束</span></span>
<span class="line"><span style="color:#E1E4E8">    } </span><span style="color:#F97583">else</span><span style="color:#E1E4E8"> {</span></span>
<span class="line"><span style="color:#6A737D">      // 5. 执行工具，把结果以 tool 角色塞回历史</span></span>
<span class="line"><span style="color:#F97583">      for</span><span style="color:#E1E4E8"> (</span><span style="color:#F97583">const</span><span style="color:#79B8FF"> toolCall</span><span style="color:#F97583"> of</span><span style="color:#E1E4E8"> toolCalls) {</span></span>
<span class="line"><span style="color:#F97583">        const</span><span style="color:#79B8FF"> result</span><span style="color:#F97583"> =</span><span style="color:#F97583"> await</span><span style="color:#79B8FF"> this</span><span style="color:#E1E4E8">.tools.</span><span style="color:#B392F0">execute</span><span style="color:#E1E4E8">(name, parseArgs);</span></span>
<span class="line"><span style="color:#79B8FF">        this</span><span style="color:#E1E4E8">.thread.</span><span style="color:#B392F0">addMessages</span><span style="color:#E1E4E8">([</span></span>
<span class="line"><span style="color:#E1E4E8">          { role: </span><span style="color:#9ECBFF">"tool"</span><span style="color:#E1E4E8">, content: result, tool_call_id: toolCall.id }</span></span>
<span class="line"><span style="color:#E1E4E8">        ]);</span></span>
<span class="line"><span style="color:#E1E4E8">      }</span></span>
<span class="line"><span style="color:#6A737D">      // 循环继续，LLM 看到工具结果后再次推理</span></span>
<span class="line"><span style="color:#E1E4E8">    }</span></span>
<span class="line"><span style="color:#E1E4E8">  }</span></span>
<span class="line"><span style="color:#E1E4E8">}</span></span></code></pre>
<p>这就是 ReAct 的核心：<strong>LLM 不直接给答案，而是输出 "我要调用某某工具"，Agent 执行工具后把结果还回去，LLM 基于结果再推理下一步</strong>。一轮不够就再来一轮，直到 LLM 说"不用工具了，这就是答案"。</p>

<h2>四、记忆系统：让 Agent 有长期记忆</h2>
<p>没有记忆的 Agent 每次对话都是白板。aura-agent 的记忆系统基于 <strong>TF-IDF + 余弦相似度</strong> 做检索，纯算法实现，不依赖向量数据库。</p>
<h3>4.1 记忆的数据结构</h3>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#6A737D">// packages/core/src/capabilities/memory/types.ts</span></span>
<span class="line"><span style="color:#F97583">export</span><span style="color:#F97583"> interface</span><span style="color:#B392F0"> MemoryEntry</span><span style="color:#E1E4E8"> {</span></span>

<span class="line"><span style="color:#FFAB70">  category</span><span style="color:#F97583">:</span><span style="color:#79B8FF"> string</span><span style="color:#E1E4E8">;      </span><span style="color:#6A737D">// 分类，如 "fact", "preference"</span></span>
<span class="line"><span style="color:#FFAB70">  importance</span><span style="color:#F97583">:</span><span style="color:#79B8FF"> number</span><span style="color:#E1E4E8">;    </span><span style="color:#6A737D">// 1-10，重要性评分</span></span>
<span class="line"><span style="color:#FFAB70">  createdAt</span><span style="color:#F97583">:</span><span style="color:#79B8FF"> string</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#FFAB70">  updatedAt</span><span style="color:#F97583">:</span><span style="color:#79B8FF"> string</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#FFAB70">  accessCount</span><span style="color:#F97583">:</span><span style="color:#79B8FF"> number</span><span style="color:#E1E4E8">;   </span><span style="color:#6A737D">// 被检索次数</span></span>

<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#6A737D">// packages/core/src/capabilities/memory/retriever.ts</span></span>

<span class="line"><span style="color:#F97583">  return</span><span style="color:#E1E4E8"> results.</span><span style="color:#B392F0">sort</span><span style="color:#E1E4E8">((</span><span style="color:#FFAB70">a</span><span style="color:#E1E4E8">, </span><span style="color:#FFAB70">b</span><span style="color:#E1E4E8">) </span><span style="color:#F97583">=></span><span style="color:#E1E4E8"> b.score </span><span style="color:#F97583">-</span><span style="color:#E1E4E8"> a.score).</span><span style="color:#B392F0">slice</span><span style="color:#E1E4E8">(</span><span style="color:#79B8FF">0</span><span style="color:#E1E4E8">, limit);</span></span>

<p>分词策略：英文按单词分，中文按字分。这样实现简单，在中小规模记忆库（几百条）上效果足够好。</p>
<h3>4.3 自动合并与智能清理</h3>
<p>保存记忆时，如果检测到相似度 > 0.8 的已有记忆，直接合并更新而不是新增：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">async </span><span style="color:#B392F0">saveMemory</span><span style="color:#E1E4E8">(content: string, category </span><span style="color:#F97583">=</span><span style="color:#9ECBFF"> "fact"</span><span style="color:#E1E4E8">, importance </span><span style="color:#F97583">=</span><span style="color:#79B8FF"> 5</span><span style="color:#E1E4E8">): </span><span style="color:#79B8FF">Promise</span><span style="color:#F97583">&#x3C;</span><span style="color:#E1E4E8">MemoryEntry</span><span style="color:#F97583">></span><span style="color:#E1E4E8"> {</span></span>
<span class="line"><span style="color:#E1E4E8">  const similar </span><span style="color:#F97583">=</span><span style="color:#B392F0"> retrieveMemories</span><span style="color:#E1E4E8">(content, </span><span style="color:#79B8FF">this</span><span style="color:#E1E4E8">.entries, </span><span style="color:#79B8FF">1</span><span style="color:#E1E4E8">, </span><span style="color:#79B8FF">0.8</span><span style="color:#E1E4E8">);</span></span>
<span class="line"><span style="color:#B392F0">  if</span><span style="color:#E1E4E8"> (similar.length > 0) {</span></span>
<span class="line"><span style="color:#F97583">    const</span><span style="color:#79B8FF"> existing</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> similar[</span><span style="color:#79B8FF">0</span><span style="color:#E1E4E8">]</span><span style="color:#F97583">!</span><span style="color:#E1E4E8">.entry;</span></span>
<span class="line"><span style="color:#E1E4E8">    existing.content </span><span style="color:#F97583">=</span><span style="color:#79B8FF"> this</span><span style="color:#E1E4E8">.</span><span style="color:#B392F0">mergeContent</span><span style="color:#E1E4E8">(existing.content, content);</span></span>
<span class="line"><span style="color:#E1E4E8">    existing.updatedAt </span><span style="color:#F97583">=</span><span style="color:#E1E4E8"> now;</span></span>
<span class="line"><span style="color:#E1E4E8">    existing.importance </span><span style="color:#F97583">=</span><span style="color:#E1E4E8"> Math.</span><span style="color:#B392F0">max</span><span style="color:#E1E4E8">(existing.importance, importance);</span></span>
<span class="line"><span style="color:#E1E4E8">    existing.accessCount</span><span style="color:#F97583">++</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#F97583">    await</span><span style="color:#79B8FF"> this</span><span style="color:#E1E4E8">.</span><span style="color:#B392F0">persist</span><span style="color:#E1E4E8">();</span></span>
<span class="line"><span style="color:#F97583">    return</span><span style="color:#E1E4E8"> existing;</span></span>
<span class="line"><span style="color:#E1E4E8">  }</span></span>
<span class="line"><span style="color:#6A737D">  // ... 创建新记忆</span></span>
<span class="line"><span style="color:#E1E4E8">}</span></span></code></pre>
<p>当记忆超过 500 条时，触发清理。评分公式：</p>
<pre><code>score = 新鲜度(30%) + 重要性(40%) + 访问频率(30%)
</code></pre>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">async </span><span style="color:#B392F0">cleanup</span><span style="color:#E1E4E8">(targetCount</span><span style="color:#F97583">?:</span><span style="color:#E1E4E8"> number): </span><span style="color:#79B8FF">Promise</span><span style="color:#F97583">&#x3C;void></span><span style="color:#E1E4E8"> {</span></span>
<span class="line"><span style="color:#E1E4E8">  const scored </span><span style="color:#F97583">=</span><span style="color:#79B8FF"> this</span><span style="color:#E1E4E8">.entries.</span><span style="color:#B392F0">map</span><span style="color:#E1E4E8">((</span><span style="color:#FFAB70">entry</span><span style="color:#E1E4E8">) </span><span style="color:#F97583">=></span><span style="color:#E1E4E8"> {</span></span>
<span class="line"><span style="color:#F97583">    const</span><span style="color:#79B8FF"> daysOld</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> (now </span><span style="color:#F97583">-</span><span style="color:#F97583"> new</span><span style="color:#B392F0"> Date</span><span style="color:#E1E4E8">(entry.createdAt).</span><span style="color:#B392F0">getTime</span><span style="color:#E1E4E8">()) </span><span style="color:#F97583">/</span><span style="color:#E1E4E8"> (</span><span style="color:#79B8FF">1000</span><span style="color:#F97583"> *</span><span style="color:#79B8FF"> 60</span><span style="color:#F97583"> *</span><span style="color:#79B8FF"> 60</span><span style="color:#F97583"> *</span><span style="color:#79B8FF"> 24</span><span style="color:#E1E4E8">);</span></span>
<span class="line"><span style="color:#F97583">    const</span><span style="color:#79B8FF"> recencyScore</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> Math.</span><span style="color:#B392F0">exp</span><span style="color:#E1E4E8">(</span><span style="color:#F97583">-</span><span style="color:#E1E4E8">daysOld </span><span style="color:#F97583">/</span><span style="color:#79B8FF"> 30</span><span style="color:#E1E4E8">);       </span><span style="color:#6A737D">// 指数衰减</span></span>
<span class="line"><span style="color:#F97583">    const</span><span style="color:#79B8FF"> importanceScore</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> entry.importance </span><span style="color:#F97583">/</span><span style="color:#79B8FF"> 10</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#F97583">    const</span><span style="color:#79B8FF"> accessScore</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> Math.</span><span style="color:#B392F0">log1p</span><span style="color:#E1E4E8">(entry.accessCount) </span><span style="color:#F97583">/</span><span style="color:#E1E4E8"> Math.</span><span style="color:#B392F0">log1p</span><span style="color:#E1E4E8">(</span><span style="color:#79B8FF">100</span><span style="color:#E1E4E8">);</span></span>
<span class="line"><span style="color:#F97583">    const</span><span style="color:#79B8FF"> totalScore</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> recencyScore </span><span style="color:#F97583">*</span><span style="color:#79B8FF"> 0.3</span><span style="color:#F97583"> +</span><span style="color:#E1E4E8"> importanceScore </span><span style="color:#F97583">*</span><span style="color:#79B8FF"> 0.4</span><span style="color:#F97583"> +</span><span style="color:#E1E4E8"> accessScore </span><span style="color:#F97583">*</span><span style="color:#79B8FF"> 0.3</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#F97583">    return</span><span style="color:#E1E4E8"> { entry, score: totalScore };</span></span>
<span class="line"><span style="color:#E1E4E8">  });</span></span>
<span class="line"></span>
<span class="line"><span style="color:#E1E4E8">  scored.sort((a, b) => b.score - a.score);</span></span>
<span class="line"><span style="color:#E1E4E8">  this.entries </span><span style="color:#F97583">=</span><span style="color:#E1E4E8"> scored.</span><span style="color:#B392F0">slice</span><span style="color:#E1E4E8">(</span><span style="color:#79B8FF">0</span><span style="color:#E1E4E8">, target).</span><span style="color:#B392F0">map</span><span style="color:#E1E4E8">((</span><span style="color:#FFAB70">s</span><span style="color:#E1E4E8">) </span><span style="color:#F97583">=></span><span style="color:#E1E4E8"> s.entry);</span></span>
<span class="line"><span style="color:#E1E4E8">}</span></span></code></pre>
<p>存储层可插拔：默认 <code>InMemoryStorage</code>，生产环境用 <code>FileMemoryStorage</code> 持久化到 JSON 文件。</p>
<h3>4.4 记忆如何注入对话</h3>
<p>Agent 在每次循环开始前，用用户查询去检索记忆，把结果塞进第一条用户消息前面：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#F97583">const</span><span style="color:#79B8FF"> memoryContext</span><span style="color:#F97583"> =</span><span style="color:#F97583"> await</span><span style="color:#79B8FF"> this</span><span style="color:#E1E4E8">.memory.</span><span style="color:#B392F0">getRelevantContext</span><span style="color:#E1E4E8">(query);</span></span>
<span class="line"><span style="color:#6A737D">// 结果格式：</span></span>

<p>这样 LLM 在推理时自然能看到相关背景，无需额外机制。</p>

<h3>6.2 Agent 层：逐块转发</h3>
<p>Agent 不缓存完整内容，而是把流式块逐块 yield 出去：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#F97583">for</span><span style="color:#F97583"> await</span><span style="color:#E1E4E8"> (</span><span style="color:#F97583">const</span><span style="color:#79B8FF"> chunk</span><span style="color:#F97583"> of</span><span style="color:#E1E4E8"> stream) {</span></span>
<span class="line"><span style="color:#F97583">  if</span><span style="color:#E1E4E8"> (chunk.type </span><span style="color:#F97583">===</span><span style="color:#9ECBFF"> "content"</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#E1E4E8">    content </span><span style="color:#F97583">+=</span><span style="color:#E1E4E8"> chunk.delta;</span></span>
<span class="line"><span style="color:#E1E4E8">    events?.</span><span style="color:#B392F0">onContentChunk</span><span style="color:#E1E4E8">?.(chunk.delta); </span><span style="color:#6A737D">// 触发外部事件</span></span>
<span class="line"><span style="color:#F97583">    yield</span><span style="color:#E1E4E8"> chunk; </span><span style="color:#6A737D">// 继续流给更外层</span></span>
<span class="line"><span style="color:#E1E4E8">  } </span><span style="color:#F97583">else</span><span style="color:#F97583"> if</span><span style="color:#E1E4E8"> (chunk.type </span><span style="color:#F97583">===</span><span style="color:#9ECBFF"> "tool_call"</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#E1E4E8">    hasToolCall </span><span style="color:#F97583">=</span><span style="color:#79B8FF"> true</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#6A737D">    // 累加工具调用参数（流式拼接）</span></span>
<span class="line"><span style="color:#F97583">    yield</span><span style="color:#E1E4E8"> chunk;</span></span>
<span class="line"><span style="color:#E1E4E8">  }</span></span>
<span class="line"><span style="color:#E1E4E8">}</span></span></code></pre>

<h2>七、Thread：对话历史与上下文截断</h2>
<p><code>Thread</code> 只负责一件事：管理消息列表和 Token 估算。</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#6A737D">// packages/core/src/agent/thread.ts</span></span>
<span class="line"><span style="color:#F97583">export</span><span style="color:#F97583"> class</span><span style="color:#B392F0"> Thread</span><span style="color:#E1E4E8"> {</span></span>
<span class="line"><span style="color:#F97583">  private</span><span style="color:#FFAB70"> messages</span><span style="color:#F97583">:</span><span style="color:#B392F0"> Message</span><span style="color:#E1E4E8">[] </span><span style="color:#F97583">=</span><span style="color:#E1E4E8"> [];</span></span>

<span class="line"><span style="color:#B392F0">  addSystemPrompt</span><span style="color:#E1E4E8">(</span><span style="color:#FFAB70">content</span><span style="color:#F97583">:</span><span style="color:#79B8FF"> string</span><span style="color:#E1E4E8">)</span><span style="color:#F97583">:</span><span style="color:#79B8FF"> void</span><span style="color:#E1E4E8"> { </span><span style="color:#F97583">...</span><span style="color:#E1E4E8"> }</span></span>
<span class="line"><span style="color:#B392F0">  addMessages</span><span style="color:#E1E4E8">(</span><span style="color:#FFAB70">messages</span><span style="color:#F97583">:</span><span style="color:#B392F0"> Message</span><span style="color:#E1E4E8">[])</span><span style="color:#F97583">:</span><span style="color:#79B8FF"> void</span><span style="color:#E1E4E8"> { </span><span style="color:#F97583">...</span><span style="color:#E1E4E8"> }</span></span>

<p>截断策略：保留 system prompt，从最早的消息开始删，但至少保留最近 2 条。这样避免上下文爆炸导致 API 费用飙升。</p>

<h2>八、架构总览</h2>
<pre><code>CLI 交互层
  └─ chatStream() → 消费 StreamChunk，实时渲染

Agent 编排层
  ├─ runLoop() → ReAct 循环，最多 10 轮
  ├─ buildMessagesWithSkills() → 动态注入技能
  ├─ Thread → 消息历史 + Token 截断
  └─ MemorySource → 检索相关记忆注入上下文

能力层
  ├─ ToolRegistry → 本地工具注册与执行
  ├─ MCPClientManager → 外部 MCP 服务器连接
  ├─ MemoryManager → TF-IDF 检索 + 自动合并/清理
  └─ SkillRegistry → 关键词匹配 + 动态注入

模型层
  └─ Provider → OpenAI / Anthropic / Gemini 统一抽象
</code></pre>

<h2>九、关键设计决策</h2>

<td>TF-IDF 而非向量数据库</td>
<td>无需外部依赖，几百条记忆上效果足够，代码自包含</td>

<td>纯 AsyncGenerator 流式</td>
<td>无中间缓存，内存友好，三层直通</td>

<td>避免 MCP 工具命名冲突，逻辑清晰</td>

<td>平衡去重与保留细节，避免记忆膨胀</td>
