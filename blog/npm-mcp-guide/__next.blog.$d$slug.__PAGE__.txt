<h2>什么是 MCP</h2>
<p>MCP（Model Context Protocol）是 Anthropic 提出的一种开放协议，用于让 AI 客户端（如 Claude Code、Claude Desktop）与外部工具和服务通信。</p>
<p>简单来说：</p>
<ul>
<li><strong>客户端</strong>：Claude Code / Claude Desktop</li>
<li><strong>服务器</strong>：你写的 MCP 服务（一个本地或远程程序）</li>
<li><strong>通信方式</strong>：JSON-RPC 2.0，通常通过 <strong>stdio</strong>（标准输入输出）进行</li>
</ul>
<p>通过 MCP，你可以让 Claude 调用自定义工具，比如：</p>
<ul>
<li>查询数据库</li>
<li>读取本地文件</li>
<li>调用图像识别 API</li>
<li>执行任意业务逻辑</li>
</ul>
<hr>
<h2>项目目标与架构</h2>
<p>我们要做一个<strong>视觉桥接 MCP 服务</strong>：</p>
<p>当 Claude Code 使用的模型无法直接理解图片时，通过这个 MCP 工具将图片发送给多模态大模型（如 Claude 3.5 Sonnet、GPT-4o、Gemini），把图片内容转换为文字描述，再返回给 Claude。</p>
<p>核心架构：</p>
<pre><code>Claude Code  &#x3C;--stdio-->  MCP Server  --HTTP-->  Multi-modal LLM API
                              |
                              v
                        describe_image
                        extract_image_text
                        ask_about_image
</code></pre>

<h3>定义工具（Tools）</h3>
<p>每个 Tool 包含：</p>
<ul>
<li><code>name</code>：工具名，Claude 会根据这个名字调用</li>
<li><code>description</code>：<strong>非常重要</strong>，Claude 通过描述决定什么时候调用这个工具</li>
<li><code>inputSchema</code>：JSON Schema，定义工具需要哪些参数</li>
</ul>
<p>例如 OCR 工具：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">{</span></span>

<span class="line"><span style="color:#B392F0">  inputSchema</span><span style="color:#E1E4E8">: {</span></span>

<span class="line"><span style="color:#B392F0">      image_url</span><span style="color:#E1E4E8">: { </span><span style="color:#B392F0">type</span><span style="color:#E1E4E8">: </span><span style="color:#9ECBFF">"string"</span><span style="color:#E1E4E8"> },</span></span>
<span class="line"><span style="color:#B392F0">      image_path</span><span style="color:#E1E4E8">: { </span><span style="color:#B392F0">type</span><span style="color:#E1E4E8">: </span><span style="color:#9ECBFF">"string"</span><span style="color:#E1E4E8"> },</span></span>

<span class="line"><span style="color:#E1E4E8">      { required: [</span><span style="color:#9ECBFF">"image_path"</span><span style="color:#E1E4E8">] },</span></span>
<span class="line"><span style="color:#E1E4E8">      { required: [</span><span style="color:#9ECBFF">"image_base64"</span><span style="color:#E1E4E8">] },</span></span>
<span class="line"><span style="color:#E1E4E8">    ],</span></span>
<span class="line"><span style="color:#E1E4E8">  },</span></span>

<h2>常见问题排查</h2>
<h3>1. <code>npx</code> 找不到包</h3>
<p>确认包已发布：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#B392F0">npm</span><span style="color:#9ECBFF"> view</span><span style="color:#9ECBFF"> @yourname/vision-bridge-mcp</span></span></code></pre>
<h3>2. MCP 服务器启动失败</h3>
<p>手动运行测试：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">API_KEY</span><span style="color:#F97583">=</span><span style="color:#9ECBFF">dummy</span><span style="color:#B392F0"> node</span><span style="color:#9ECBFF"> dist/index.js</span></span></code></pre>
<p>如果没有错误输出，说明服务器能正常启动。</p>
<h3>3. Claude 不调用工具</h3>
<ul>
<li>检查工具 <code>description</code> 是否清晰</li>
<li>检查 <code>inputSchema</code> 是否正确</li>
<li>在 <code>.claude/CLAUDE.md</code> 中明确提示何时调用工具</li>
</ul>
<h3>4. API 返回 401</h3>
<ul>
<li>Anthropic：确认 <code>x-api-key</code> 和 <code>anthropic-version</code> 头</li>
<li>OpenAI：确认 <code>Authorization: Bearer</code> 头</li>
<li>Gemini：确认 key 作为 URL query parameter</li>
</ul>
<h3>5. 发布到 npm 时 403</h3>
<ul>
<li>包名已被占用：改用 scope（<code>@yourname/xxx</code>）</li>
<li>2FA 限制：使用 access token 或 <code>--otp</code></li>
<li>未登录：先 <code>npm login</code></li>
</ul>
