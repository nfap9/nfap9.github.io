21:T14746,<h1>跨域（Cross-Origin）详解与解决方案</h1>
<h2>一、什么是跨域？</h2>
<h3>1.1 同源策略（Same-Origin Policy）</h3>
<p><strong>同源策略</strong>是浏览器最核心的安全策略之一，由 Netscape 公司于 1995 年引入。它限制了一个源（origin）的文档或脚本如何与另一个源的资源进行交互。这是一个用于隔离潜在恶意文件的重要安全机制。</p>
<blockquote>
<p><strong>同源的定义</strong>：如果两个 URL 的**协议（protocol）、域名（host）、端口（port）**都相同，则它们是同源的。</p>
</blockquote>
<table>
<thead>
<tr>
<th align="left">URL</th>
<th align="left">与 <code>http://www.example.com:80/page.html</code> 是否同源</th>
<th align="left">原因</th>
</tr>
</thead>
<tbody>
<tr>
<td align="left"><code>http://www.example.com:80/page2.html</code></td>
<td align="left">同源</td>
<td align="left">协议、域名、端口均相同</td>
</tr>
<tr>
<td align="left"><code>http://www.example.com/page2.html</code></td>
<td align="left">同源</td>
<td align="left">默认端口 80 可省略</td>
</tr>
<tr>
<td align="left"><code>https://www.example.com/page.html</code></td>
<td align="left">跨域</td>
<td align="left">协议不同（http vs https）</td>
</tr>
<tr>
<td align="left"><code>http://api.example.com/page.html</code></td>
<td align="left">跨域</td>
<td align="left">域名不同（子域不同）</td>
</tr>
<tr>
<td align="left"><code>http://www.example.com:8080/page.html</code></td>
<td align="left">跨域</td>
<td align="left">端口不同（80 vs 8080）</td>
</tr>

<td align="left">跨域</td>
<td align="left">域名完全不同</td>
</tr>

<h3>1.2 跨域的场景</h3>
<p>浏览器在执行以下操作时，会受到同源策略的限制：</p>
<ul>
<li><strong>AJAX/Fetch 请求</strong>：XMLHttpRequest 和 Fetch API 默认只能请求同源资源</li>
<li><strong>Cookie、LocalStorage、IndexedDB</strong>：无法读取非同源的存储数据</li>
<li><strong>DOM 操作</strong>：无法获取非同源页面的 DOM 节点</li>
<li><strong>Canvas/WebGL</strong>：使用跨域图片时可能被污染（tainted canvas）</li>
</ul>
<blockquote>
<p><strong>注意</strong>：跨域限制是<strong>浏览器的行为</strong>，服务器之间或命令行工具（如 curl、Postman）请求不受同源策略限制。</p>
</blockquote>
<hr>
<h2>二、AJAX 跨域解决方法</h2>
<h3>2.1 JSONP（JSON with Padding）</h3>
<p>JSONP 是一种利用 <code>&#x3C;script></code> 标签不受同源策略限制特性的跨域解决方案。</p>
<p><strong>原理</strong>：</p>
<ul>
<li><code>&#x3C;script></code>、<code>&#x3C;img></code>、<code>&#x3C;link></code> 等标签的 <code>src</code> 属性可以加载跨域资源</li>
<li>服务端将数据包装在一个回调函数中返回</li>
<li>浏览器执行该回调函数，获取数据</li>
</ul>

<h3>2.2 跨域资源共享（CORS）</h3>
<p>CORS（Cross-Origin Resource Sharing）是 W3C 标准，目前最主流的跨域解决方案。它允许服务器通过设置特定的 HTTP 响应头来告知浏览器是否允许跨域请求。</p>
<h4>CORS 请求分类</h4>
<p><strong>（1）简单请求（Simple Request）</strong></p>
<p>需同时满足以下条件：</p>
<ul>
<li>方法为 <code>GET</code>、<code>HEAD</code>、<code>POST</code> 之一</li>
<li>头部字段仅限于：<code>Accept</code>、<code>Accept-Language</code>、<code>Content-Language</code>、<code>Content-Type</code></li>
<li><code>Content-Type</code> 仅限于：<code>application/x-www-form-urlencoded</code>、<code>multipart/form-data</code>、<code>text/plain</code></li>
</ul>
<p>简单请求流程：</p>
<pre><code>浏览器直接发送请求，自动在请求头中添加 Origin 字段
服务器返回 Access-Control-Allow-Origin 响应头
浏览器判断该响应头是否允许当前源访问
</code></pre>
<p><strong>（2）预检请求（Preflight Request）</strong></p>
<p>对于非简单请求，浏览器会先发送 <code>OPTIONS</code> 预检请求：</p>
<pre><code>OPTIONS /api/data HTTP/1.1
Origin: http://example.com
Access-Control-Request-Method: PUT
Access-Control-Request-Headers: Content-Type,Authorization
</code></pre>
<p>服务端响应：</p>
<pre><code>HTTP/1.1 204 No Content
Access-Control-Allow-Origin: http://example.com
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Access-Control-Allow-Headers: Content-Type, Authorization
Access-Control-Max-Age: 86400  // 预检结果缓存时间（秒）
</code></pre>

<h3>2.4 服务端代理（反向代理）</h3>
<p>通过在同源的服务器端设置代理，让服务器代替浏览器访问跨域资源，再将结果返回给浏览器。浏览器始终只与同源的代理服务器通信，完美避开跨域限制。</p>
<p><strong>实现方式</strong>：</p>
<p><strong>（1）开发环境代理（Webpack DevServer）</strong>：</p>

<span class="line"><span style="color:#E1E4E8">  devServer: {</span></span>
<span class="line"><span style="color:#E1E4E8">    proxy: {</span></span>
<span class="line"><span style="color:#9ECBFF">      '/api'</span><span style="color:#E1E4E8">: {</span></span>
<span class="line"><span style="color:#E1E4E8">        target: </span><span style="color:#9ECBFF">'https://api.example.com'</span><span style="color:#E1E4E8">,  </span><span style="color:#6A737D">// 目标 API 服务器</span></span>
<span class="line"><span style="color:#E1E4E8">        changeOrigin: </span><span style="color:#79B8FF">true</span><span style="color:#E1E4E8">,  </span><span style="color:#6A737D">// 改变请求源头</span></span>
<span class="line"><span style="color:#E1E4E8">        pathRewrite: {</span></span>
<span class="line"><span style="color:#9ECBFF">          '^/api'</span><span style="color:#E1E4E8">: </span><span style="color:#9ECBFF">''</span><span style="color:#6A737D">  // 重写路径：/api/users → /users</span></span>
<span class="line"><span style="color:#E1E4E8">        },</span></span>
<span class="line"><span style="color:#E1E4E8">        secure: </span><span style="color:#79B8FF">false</span><span style="color:#E1E4E8">,  </span><span style="color:#6A737D">// 接受 https 自签名证书</span></span>
<span class="line"><span style="color:#6A737D">        // 可选：添加自定义请求头</span></span>
<span class="line"><span style="color:#E1E4E8">        headers: {</span></span>

<h2>三、前端跨域通信（iframe 场景）</h2>
<p>当页面中使用 <code>&#x3C;iframe></code> 嵌入不同源的页面时，两者之间直接的 JavaScript 访问会受到同源策略限制。以下是几种解决方案：</p>
<h3>3.1 document.domain + iframe（仅限子域）</h3>
<p><strong>适用场景</strong>：主域相同，仅子域不同。如 <code>a.example.com</code> 和 <code>b.example.com</code>。</p>
<p><strong>原理</strong>：将两个页面的 <code>document.domain</code> 都设置为主域 <code>example.com</code>，浏览器即认为它们同源。</p>
<p><strong>代码示例</strong>：</p>
<p><code>http://a.example.com/page.html</code>：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">&#x3C;!</span><span style="color:#85E89D">DOCTYPE</span><span style="color:#B392F0"> html</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">html</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">body</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">  &#x3C;</span><span style="color:#85E89D">iframe</span><span style="color:#B392F0"> id</span><span style="color:#E1E4E8">=</span><span style="color:#9ECBFF">"iframe"</span><span style="color:#B392F0"> src</span><span style="color:#E1E4E8">=</span><span style="color:#9ECBFF">"http://b.example.com/page.html"</span><span style="color:#E1E4E8">>&#x3C;/</span><span style="color:#85E89D">iframe</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">  &#x3C;</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#6A737D">    // 必须双方都设置</span></span>
<span class="line"><span style="color:#E1E4E8">    document.domain </span><span style="color:#F97583">=</span><span style="color:#9ECBFF"> 'example.com'</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#E1E4E8">    </span></span>
<span class="line"><span style="color:#E1E4E8">    window.</span><span style="color:#B392F0">onload</span><span style="color:#F97583"> =</span><span style="color:#F97583"> function</span><span style="color:#E1E4E8">() {</span></span>
<span class="line"><span style="color:#F97583">      const</span><span style="color:#79B8FF"> iframe</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> document.</span><span style="color:#B392F0">getElementById</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'iframe'</span><span style="color:#E1E4E8">);</span></span>
<span class="line"><span style="color:#6A737D">      // 现在可以访问 iframe 的内容</span></span>
<span class="line"><span style="color:#E1E4E8">      console.</span><span style="color:#B392F0">log</span><span style="color:#E1E4E8">(iframe.contentWindow.document.body.innerHTML);</span></span>
<span class="line"><span style="color:#E1E4E8">    };</span></span>
<span class="line"><span style="color:#E1E4E8">  &#x3C;/</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;/</span><span style="color:#85E89D">body</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;/</span><span style="color:#85E89D">html</span><span style="color:#E1E4E8">></span></span></code></pre>
<p><code>http://b.example.com/page.html</code>：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">&#x3C;!</span><span style="color:#85E89D">DOCTYPE</span><span style="color:#B392F0"> html</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">html</span><span style="color:#E1E4E8">></span></span>

<span class="line"><span style="color:#E1E4E8">  &#x3C;/</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;/</span><span style="color:#85E89D">body</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;/</span><span style="color:#85E89D">html</span><span style="color:#E1E4E8">></span></span></code></pre>
<blockquote>
<p><strong>限制</strong>：</p>
<ul>
<li>只能用于<strong>主域相同、子域不同</strong>的场景</li>
<li>设置 <code>document.domain</code> 后<strong>不可逆</strong>，且只能设置为当前域的父域</li>
<li>现代浏览器中此方法逐渐被淘汰，建议优先使用 <code>postMessage</code></li>
</ul>

<h3>3.2 location.hash + iframe</h3>
<p><strong>原理</strong>：利用 <code>location.hash</code> 的改变不会触发页面刷新，且可以被同域页面读取的特性。通过中间代理页实现跨域通信。</p>
<p><strong>架构</strong>：</p>
<pre><code>A页面 (a.com) 
  → iframe 嵌套 B页面 (b.com)
    → B页面 iframe 嵌套 C页面 (a.com/proxy.html)  // C与A同源
</code></pre>
<p><strong>代码示例</strong>：</p>
<p><code>http://a.com/a.html</code>（父页面）：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">iframe</span><span style="color:#B392F0"> id</span><span style="color:#E1E4E8">=</span><span style="color:#9ECBFF">"iframe"</span><span style="color:#B392F0"> src</span><span style="color:#E1E4E8">=</span><span style="color:#9ECBFF">"http://b.com/b.html"</span><span style="color:#E1E4E8">>&#x3C;/</span><span style="color:#85E89D">iframe</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#6A737D">  // 1. 向 B 页面传递数据</span></span>
<span class="line"><span style="color:#F97583">  const</span><span style="color:#79B8FF"> iframe</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> document.</span><span style="color:#B392F0">getElementById</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'iframe'</span><span style="color:#E1E4E8">);</span></span>
<span class="line"><span style="color:#E1E4E8">  </span></span>
<span class="line"><span style="color:#F97583">  function</span><span style="color:#B392F0"> sendMessageToB</span><span style="color:#E1E4E8">(</span><span style="color:#FFAB70">data</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#6A737D">    // 通过修改 B 页面的 hash 来传递数据</span></span>
<span class="line"><span style="color:#E1E4E8">    iframe.src </span><span style="color:#F97583">=</span><span style="color:#9ECBFF"> `http://b.com/b.html#${</span><span style="color:#B392F0">encodeURIComponent</span><span style="color:#9ECBFF">(</span><span style="color:#79B8FF">JSON</span><span style="color:#9ECBFF">.</span><span style="color:#B392F0">stringify</span><span style="color:#9ECBFF">(</span><span style="color:#E1E4E8">data</span><span style="color:#9ECBFF">))</span><span style="color:#9ECBFF">}`</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#E1E4E8">  }</span></span>
<span class="line"><span style="color:#E1E4E8">  </span></span>
<span class="line"><span style="color:#6A737D">  // 2. 监听 C 代理页面传回的数据</span></span>
<span class="line"><span style="color:#F97583">  function</span><span style="color:#B392F0"> checkHash</span><span style="color:#E1E4E8">() {</span></span>
<span class="line"><span style="color:#F97583">    const</span><span style="color:#79B8FF"> hash</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> location.hash;</span></span>
<span class="line"><span style="color:#F97583">    if</span><span style="color:#E1E4E8"> (hash </span><span style="color:#F97583">&#x26;&#x26;</span><span style="color:#E1E4E8"> hash.</span><span style="color:#79B8FF">length</span><span style="color:#F97583"> ></span><span style="color:#79B8FF"> 1</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#F97583">      const</span><span style="color:#79B8FF"> data</span><span style="color:#F97583"> =</span><span style="color:#79B8FF"> JSON</span><span style="color:#E1E4E8">.</span><span style="color:#B392F0">parse</span><span style="color:#E1E4E8">(</span><span style="color:#B392F0">decodeURIComponent</span><span style="color:#E1E4E8">(hash.</span><span style="color:#B392F0">substring</span><span style="color:#E1E4E8">(</span><span style="color:#79B8FF">1</span><span style="color:#E1E4E8">)));</span></span>
<span class="line"><span style="color:#E1E4E8">      console.</span><span style="color:#B392F0">log</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'收到 B 页面的回复:'</span><span style="color:#E1E4E8">, data);</span></span>
<span class="line"><span style="color:#E1E4E8">      location.hash </span><span style="color:#F97583">=</span><span style="color:#9ECBFF"> ''</span><span style="color:#E1E4E8">;  </span><span style="color:#6A737D">// 清理 hash</span></span>
<span class="line"><span style="color:#E1E4E8">    }</span></span>
<span class="line"><span style="color:#E1E4E8">  }</span></span>
<span class="line"><span style="color:#E1E4E8">  </span></span>
<span class="line"><span style="color:#B392F0">  setInterval</span><span style="color:#E1E4E8">(checkHash, </span><span style="color:#79B8FF">100</span><span style="color:#E1E4E8">);  </span><span style="color:#6A737D">// 轮询监听 hash 变化</span></span>
<span class="line"><span style="color:#B392F0">  sendMessageToB</span><span style="color:#E1E4E8">({ action: </span><span style="color:#9ECBFF">'getUserInfo'</span><span style="color:#E1E4E8">, userId: </span><span style="color:#79B8FF">123</span><span style="color:#E1E4E8"> });</span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;/</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span></code></pre>
<p><code>http://b.com/b.html</code>（跨域子页面）：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span>

<span class="line"><span style="color:#E1E4E8">    console.</span><span style="color:#B392F0">log</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'收到 A 页面的消息:'</span><span style="color:#E1E4E8">, data);</span></span>
<span class="line"><span style="color:#E1E4E8">    </span></span>
<span class="line"><span style="color:#6A737D">    // 2. 处理数据后，通过同域的 C 页面将结果传回 A</span></span>
<span class="line"><span style="color:#F97583">    const</span><span style="color:#79B8FF"> result</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> { status: </span><span style="color:#9ECBFF">'success'</span><span style="color:#E1E4E8">, userInfo: { name: </span><span style="color:#9ECBFF">'张三'</span><span style="color:#E1E4E8">, age: </span><span style="color:#79B8FF">25</span><span style="color:#E1E4E8"> } };</span></span>
<span class="line"><span style="color:#E1E4E8">    </span></span>
<span class="line"><span style="color:#6A737D">    // C 页面与 A 页面同源（都在 a.com 下）</span></span>
<span class="line"><span style="color:#F97583">    const</span><span style="color:#79B8FF"> proxyIframe</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> document.</span><span style="color:#B392F0">createElement</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'iframe'</span><span style="color:#E1E4E8">);</span></span>

<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span>

<span class="line"><span style="color:#F97583">  const</span><span style="color:#79B8FF"> hash</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> location.hash;</span></span>
<span class="line"><span style="color:#F97583">  if</span><span style="color:#E1E4E8"> (window.parent </span><span style="color:#F97583">!==</span><span style="color:#E1E4E8"> window) {</span></span>
<span class="line"><span style="color:#E1E4E8">    window.parent.parent.location.hash </span><span style="color:#F97583">=</span><span style="color:#E1E4E8"> hash;  </span><span style="color:#6A737D">// 跨两层 iframe 传给 A</span></span>
<span class="line"><span style="color:#E1E4E8">  }</span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;/</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span></code></pre>
<blockquote>
<p><strong>缺点</strong>：</p>
<ul>
<li>实现复杂，需要三个页面配合</li>
<li>使用轮询监听 hash，性能较差</li>
<li>数据暴露在 URL 中，有长度限制且不安全</li>
<li><strong>已过时</strong>，仅作为了解</li>
</ul>
</blockquote>
<h3>3.3 window.name</h3>
<p><strong>原理</strong>：<code>window.name</code> 属性有一个特殊特性——页面跳转后它的值保持不变，且没有同源限制。利用这一特性可以实现跨域数据传递。</p>
<p><strong>代码示例</strong>：</p>
<p><code>http://a.com/a.html</code>（父页面）：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">iframe</span><span style="color:#B392F0"> id</span><span style="color:#E1E4E8">=</span><span style="color:#9ECBFF">"iframe"</span><span style="color:#B392F0"> src</span><span style="color:#E1E4E8">=</span><span style="color:#9ECBFF">"http://b.com/b.html"</span><span style="color:#E1E4E8">>&#x3C;/</span><span style="color:#85E89D">iframe</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#F97583">  let</span><span style="color:#E1E4E8"> state </span><span style="color:#F97583">=</span><span style="color:#79B8FF"> 0</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#F97583">  const</span><span style="color:#79B8FF"> iframe</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> document.</span><span style="color:#B392F0">getElementById</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'iframe'</span><span style="color:#E1E4E8">);</span></span>
<span class="line"><span style="color:#E1E4E8">  </span></span>
<span class="line"><span style="color:#E1E4E8">  iframe.</span><span style="color:#B392F0">onload</span><span style="color:#F97583"> =</span><span style="color:#F97583"> function</span><span style="color:#E1E4E8">() {</span></span>
<span class="line"><span style="color:#F97583">    if</span><span style="color:#E1E4E8"> (state </span><span style="color:#F97583">===</span><span style="color:#79B8FF"> 0</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#6A737D">      // 第一次加载：B 页面正在设置 window.name</span></span>
<span class="line"><span style="color:#6A737D">      // 将 iframe 指向同源的代理页面，以便读取 name</span></span>
<span class="line"><span style="color:#E1E4E8">      iframe.src </span><span style="color:#F97583">=</span><span style="color:#9ECBFF"> 'http://a.com/proxy.html'</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#E1E4E8">      state </span><span style="color:#F97583">=</span><span style="color:#79B8FF"> 1</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#E1E4E8">    } </span><span style="color:#F97583">else</span><span style="color:#F97583"> if</span><span style="color:#E1E4E8"> (state </span><span style="color:#F97583">===</span><span style="color:#79B8FF"> 1</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#6A737D">      // 第二次加载：现在可以读取到 window.name 了</span></span>
<span class="line"><span style="color:#E1E4E8">      console.</span><span style="color:#B392F0">log</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'获取到数据:'</span><span style="color:#E1E4E8">, iframe.contentWindow.name);</span></span>
<span class="line"><span style="color:#6A737D">      // 数据使用完后清理</span></span>
<span class="line"><span style="color:#E1E4E8">      iframe.contentWindow.name </span><span style="color:#F97583">=</span><span style="color:#9ECBFF"> ''</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#E1E4E8">    }</span></span>
<span class="line"><span style="color:#E1E4E8">  };</span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;/</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span></code></pre>
<p><code>http://b.com/b.html</code>（数据页面）：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#6A737D">  // 将要传递的数据写入 window.name</span></span>
<span class="line"><span style="color:#E1E4E8">  window.name </span><span style="color:#F97583">=</span><span style="color:#79B8FF"> JSON</span><span style="color:#E1E4E8">.</span><span style="color:#B392F0">stringify</span><span style="color:#E1E4E8">({</span></span>
<span class="line"><span style="color:#E1E4E8">    users: [</span></span>
<span class="line"><span style="color:#E1E4E8">      { id: </span><span style="color:#79B8FF">1</span><span style="color:#E1E4E8">, name: </span><span style="color:#9ECBFF">'张三'</span><span style="color:#E1E4E8"> },</span></span>
<span class="line"><span style="color:#E1E4E8">      { id: </span><span style="color:#79B8FF">2</span><span style="color:#E1E4E8">, name: </span><span style="color:#9ECBFF">'李四'</span><span style="color:#E1E4E8"> }</span></span>
<span class="line"><span style="color:#E1E4E8">    ],</span></span>
<span class="line"><span style="color:#E1E4E8">    total: </span><span style="color:#79B8FF">100</span></span>
<span class="line"><span style="color:#E1E4E8">  });</span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;/</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span></code></pre>
<p><code>http://a.com/proxy.html</code>（空白代理页，仅用于切换同源）：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">&#x3C;!</span><span style="color:#85E89D">DOCTYPE</span><span style="color:#B392F0"> html</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">html</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">body</span><span style="color:#E1E4E8">>&#x3C;/</span><span style="color:#85E89D">body</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;/</span><span style="color:#85E89D">html</span><span style="color:#E1E4E8">></span></span></code></pre>
<blockquote>
<p><strong>缺点</strong>：</p>
<ul>
<li>需要 iframe 两次加载，体验不佳</li>
<li><code>window.name</code> 只能存储字符串，且有大小限制（约 2MB）</li>
<li>安全性较低，其他脚本也能修改 <code>window.name</code></li>
<li><strong>已过时</strong>，建议用 <code>postMessage</code> 替代</li>
</ul>
</blockquote>
<h3>3.4 postMessage（推荐）</h3>
<p><code>postMessage</code> 是 HTML5 引入的现代跨域通信 API，是目前<strong>最推荐</strong>的跨域通信方案。它安全、简洁、功能强大。</p>
<p><strong>API 说明</strong>：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#6A737D">// 发送消息</span></span>
<span class="line"><span style="color:#E1E4E8">otherWindow.</span><span style="color:#B392F0">postMessage</span><span style="color:#E1E4E8">(message, targetOrigin, [transfer]);</span></span>
<span class="line"></span>
<span class="line"><span style="color:#6A737D">/**</span></span>
<span class="line"><span style="color:#6A737D"> * </span><span style="color:#F97583">@param</span><span style="color:#B392F0"> {any}</span><span style="color:#E1E4E8"> message</span><span style="color:#6A737D"> - 要发送的数据（会被结构化克隆算法序列化）</span></span>
<span class="line"><span style="color:#6A737D"> * </span><span style="color:#F97583">@param</span><span style="color:#B392F0"> {string}</span><span style="color:#E1E4E8"> targetOrigin</span><span style="color:#6A737D"> - 目标窗口的源，如 'http://example.com'，'*' 表示不限制</span></span>
<span class="line"><span style="color:#6A737D"> * </span><span style="color:#F97583">@param</span><span style="color:#B392F0"> {Transferable[]}</span><span style="color:#E1E4E8"> [transfer]</span><span style="color:#6A737D"> - 可选，可转移对象</span></span>
<span class="line"><span style="color:#6A737D"> */</span></span>
<span class="line"></span>
<span class="line"><span style="color:#6A737D">// 接收消息</span></span>
<span class="line"><span style="color:#E1E4E8">window.</span><span style="color:#B392F0">addEventListener</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'message'</span><span style="color:#E1E4E8">, </span><span style="color:#F97583">function</span><span style="color:#E1E4E8">(</span><span style="color:#FFAB70">event</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#6A737D">  // event.source - 发送消息的窗口引用</span></span>
<span class="line"><span style="color:#6A737D">  // event.origin - 发送消息的源（协议+域名+端口）</span></span>
<span class="line"><span style="color:#6A737D">  // event.data - 接收到的数据</span></span>
<span class="line"><span style="color:#E1E4E8">});</span></span></code></pre>
<p><strong>代码示例</strong>：</p>
<p><code>http://a.com/parent.html</code>（父页面）：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">iframe</span><span style="color:#B392F0"> id</span><span style="color:#E1E4E8">=</span><span style="color:#9ECBFF">"iframe"</span><span style="color:#B392F0"> src</span><span style="color:#E1E4E8">=</span><span style="color:#9ECBFF">"http://b.com/child.html"</span><span style="color:#E1E4E8">>&#x3C;/</span><span style="color:#85E89D">iframe</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span>
<span class="line"><span style="color:#F97583">  const</span><span style="color:#79B8FF"> iframe</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> document.</span><span style="color:#B392F0">getElementById</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'iframe'</span><span style="color:#E1E4E8">);</span></span>
<span class="line"><span style="color:#E1E4E8">  </span></span>
<span class="line"><span style="color:#6A737D">  // 等待 iframe 加载完成</span></span>
<span class="line"><span style="color:#E1E4E8">  iframe.</span><span style="color:#B392F0">onload</span><span style="color:#F97583"> =</span><span style="color:#F97583"> function</span><span style="color:#E1E4E8">() {</span></span>
<span class="line"><span style="color:#6A737D">    // 向子页面发送消息</span></span>
<span class="line"><span style="color:#E1E4E8">    iframe.contentWindow.</span><span style="color:#B392F0">postMessage</span><span style="color:#E1E4E8">(</span></span>
<span class="line"><span style="color:#E1E4E8">      { type: </span><span style="color:#9ECBFF">'REQUEST_USER_INFO'</span><span style="color:#E1E4E8">, userId: </span><span style="color:#79B8FF">123</span><span style="color:#E1E4E8"> },</span></span>
<span class="line"><span style="color:#9ECBFF">      'http://b.com'</span><span style="color:#6A737D">  // 明确指定目标源，不要使用 *</span></span>
<span class="line"><span style="color:#E1E4E8">    );</span></span>
<span class="line"><span style="color:#E1E4E8">  };</span></span>
<span class="line"><span style="color:#E1E4E8">  </span></span>
<span class="line"><span style="color:#6A737D">  // 监听子页面的回复</span></span>
<span class="line"><span style="color:#E1E4E8">  window.</span><span style="color:#B392F0">addEventListener</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'message'</span><span style="color:#E1E4E8">, </span><span style="color:#F97583">function</span><span style="color:#E1E4E8">(</span><span style="color:#FFAB70">event</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#6A737D">    // 必须验证消息来源！</span></span>
<span class="line"><span style="color:#F97583">    if</span><span style="color:#E1E4E8"> (event.origin </span><span style="color:#F97583">!==</span><span style="color:#9ECBFF"> 'http://b.com'</span><span style="color:#E1E4E8">) </span><span style="color:#F97583">return</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#E1E4E8">    </span></span>
<span class="line"><span style="color:#6A737D">    // 验证消息类型</span></span>
<span class="line"><span style="color:#F97583">    if</span><span style="color:#E1E4E8"> (event.data.type </span><span style="color:#F97583">===</span><span style="color:#9ECBFF"> 'USER_INFO_RESPONSE'</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#E1E4E8">      console.</span><span style="color:#B392F0">log</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'收到用户信息:'</span><span style="color:#E1E4E8">, event.data.payload);</span></span>
<span class="line"><span style="color:#E1E4E8">    }</span></span>
<span class="line"><span style="color:#E1E4E8">  });</span></span>
<span class="line"><span style="color:#E1E4E8">&#x3C;/</span><span style="color:#85E89D">script</span><span style="color:#E1E4E8">></span></span></code></pre>
<p><code>http://b.com/child.html</code>（iframe 子页面）：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#6A737D">// 监听父页面的消息</span></span>
<span class="line"><span style="color:#E1E4E8">window.</span><span style="color:#B392F0">addEventListener</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'message'</span><span style="color:#E1E4E8">, </span><span style="color:#F97583">function</span><span style="color:#E1E4E8">(</span><span style="color:#FFAB70">event</span><span style="color:#E1E4E8">) {</span></span>

<span class="line"><span style="color:#E1E4E8">    console.</span><span style="color:#B392F0">warn</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'收到来自未知源的消息:'</span><span style="color:#E1E4E8">, event.origin);</span></span>

<span class="line"><span style="color:#E1E4E8">  </span></span>
<span class="line"><span style="color:#F97583">  const</span><span style="color:#E1E4E8"> { </span><span style="color:#79B8FF">type</span><span style="color:#E1E4E8">, </span><span style="color:#79B8FF">userId</span><span style="color:#E1E4E8"> } </span><span style="color:#F97583">=</span><span style="color:#E1E4E8"> event.data;</span></span>
<span class="line"><span style="color:#E1E4E8">  </span></span>
<span class="line"><span style="color:#F97583">  if</span><span style="color:#E1E4E8"> (type </span><span style="color:#F97583">===</span><span style="color:#9ECBFF"> 'REQUEST_USER_INFO'</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#6A737D">    // 处理请求</span></span>
<span class="line"><span style="color:#F97583">    const</span><span style="color:#79B8FF"> userInfo</span><span style="color:#F97583"> =</span><span style="color:#B392F0"> getUserInfo</span><span style="color:#E1E4E8">(userId);</span></span>
<span class="line"><span style="color:#E1E4E8">    </span></span>
<span class="line"><span style="color:#6A737D">    // 回复父页面</span></span>
<span class="line"><span style="color:#E1E4E8">    event.source.</span><span style="color:#B392F0">postMessage</span><span style="color:#E1E4E8">(</span></span>
<span class="line"><span style="color:#E1E4E8">      { </span></span>
<span class="line"><span style="color:#E1E4E8">        type: </span><span style="color:#9ECBFF">'USER_INFO_RESPONSE'</span><span style="color:#E1E4E8">, </span></span>
<span class="line"><span style="color:#E1E4E8">        payload: userInfo </span></span>
<span class="line"><span style="color:#E1E4E8">      },</span></span>
<span class="line"><span style="color:#E1E4E8">      event.origin  </span><span style="color:#6A737D">// 使用 event.origin 确保回复到正确的源</span></span>
<span class="line"><span style="color:#E1E4E8">    );</span></span>
<span class="line"><span style="color:#E1E4E8">  }</span></span>
<span class="line"><span style="color:#E1E4E8">});</span></span>
<span class="line"></span>
<span class="line"><span style="color:#F97583">function</span><span style="color:#B392F0"> getUserInfo</span><span style="color:#E1E4E8">(</span><span style="color:#FFAB70">id</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#F97583">  return</span><span style="color:#E1E4E8"> { id, name: </span><span style="color:#9ECBFF">'张三'</span><span style="color:#E1E4E8">, email: </span><span style="color:#9ECBFF">'zhangsan@example.com'</span><span style="color:#E1E4E8"> };</span></span>
<span class="line"><span style="color:#E1E4E8">}</span></span></code></pre>
<p><strong><code>postMessage</code> 安全最佳实践</strong>：</p>
<pre class="shiki github-dark" style="background-color:#24292e;color:#e1e4e8" tabindex="0"><code><span class="line"><span style="color:#E1E4E8">window.</span><span style="color:#B392F0">addEventListener</span><span style="color:#E1E4E8">(</span><span style="color:#9ECBFF">'message'</span><span style="color:#E1E4E8">, </span><span style="color:#F97583">function</span><span style="color:#E1E4E8">(</span><span style="color:#FFAB70">event</span><span style="color:#E1E4E8">) {</span></span>
<span class="line"><span style="color:#6A737D">  // 1. 始终验证消息来源</span></span>
<span class="line"><span style="color:#F97583">  if</span><span style="color:#E1E4E8"> (</span><span style="color:#F97583">!</span><span style="color:#E1E4E8">[</span><span style="color:#9ECBFF">'https://trusted-parent.com'</span><span style="color:#E1E4E8">, </span><span style="color:#9ECBFF">'https://partner.com'</span><span style="color:#E1E4E8">].</span><span style="color:#B392F0">includes</span><span style="color:#E1E4E8">(event.origin)) {</span></span>
<span class="line"><span style="color:#F97583">    return</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#E1E4E8">  }</span></span>
<span class="line"><span style="color:#E1E4E8">  </span></span>

<span class="line"><span style="color:#E1E4E8">  </span></span>
<span class="line"><span style="color:#6A737D">  // 3. 使用白名单校验消息类型</span></span>
<span class="line"><span style="color:#F97583">  const</span><span style="color:#79B8FF"> ALLOWED_TYPES</span><span style="color:#F97583"> =</span><span style="color:#E1E4E8"> [</span><span style="color:#9ECBFF">'USER_INFO'</span><span style="color:#E1E4E8">, </span><span style="color:#9ECBFF">'SETTINGS'</span><span style="color:#E1E4E8">, </span><span style="color:#9ECBFF">'LOGOUT'</span><span style="color:#E1E4E8">];</span></span>
<span class="line"><span style="color:#F97583">  if</span><span style="color:#E1E4E8"> (</span><span style="color:#F97583">!</span><span style="color:#79B8FF">ALLOWED_TYPES</span><span style="color:#E1E4E8">.</span><span style="color:#B392F0">includes</span><span style="color:#E1E4E8">(event.data.type)) </span><span style="color:#F97583">return</span><span style="color:#E1E4E8">;</span></span>
<span class="line"><span style="color:#E1E4E8">  </span></span>
<span class="line"><span style="color:#6A737D">  // 4. 处理消息</span></span>
<span class="line"><span style="color:#B392F0">  handleMessage</span><span style="color:#E1E4E8">(event.data, event.source, event.origin);</span></span>
<span class="line"><span style="color:#E1E4E8">});</span></span></code></pre>

<h2>六、安全注意事项</h2>
<ol>
<li>
<p><strong>CORS 不是安全机制</strong>：CORS 只是浏览器的访问控制机制，不能替代身份认证和权限校验。服务端仍需验证用户身份。</p>
</li>
<li>
<p>**不要滥用 <code>Access-Control-Allow-Origin: ***</code>：在生产环境中，尽量明确指定允许的域名，尤其是涉及敏感数据或需要携带 Cookie 时。</p>
</li>
<li>
<p><strong><code>postMessage</code> 必须验证 <code>origin</code></strong>：接收方一定要检查 <code>event.origin</code>，避免处理来自恶意网站的消息。</p>
</li>
<li>
<p><strong>避免使用已废弃的方案</strong>：JSONP、<code>location.hash</code>、<code>window.name</code> 等方法存在安全风险或兼容性问题，新项目应避免使用。</p>
</li>
<li>
<p><strong>使用 HTTPS</strong>：生产环境务必启用 HTTPS，防止中间人攻击篡改 CORS 头部。</p>
</li>
</ol>
