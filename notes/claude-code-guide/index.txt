<h3>ccswitch 安装与配置</h3>
<p>推荐用 <code>ccswitch</code> 管理多个模型配置，一键切换。</p>
<pre><code class="language-bash"># 安装
npm install -g claude-settings-switch

# 添加 GLM 配置
ccs add glm https://open.bigmodel.cn/api/paas/v4/ &#x3C;glm-api-key>

# 添加 Kimi 配置
ccs add kimi https://api.moonshot.cn/v1 &#x3C;kimi-api-key>

# 添加 DeepSeek 配置
ccs add ds https://api.deepseek.com/v1 &#x3C;deepseek-api-key>

# 查看已有配置
ccs list

# 切换到 GLM
ccs sw glm

# 查看当前使用的配置
ccs current
</code></pre>
<p>配置文件存储在 <code>~/.ccswitcher/configs.json</code>，Claude Code 配置在 <code>~/.claude/settings.json</code>。</p>
<h3>手动配置（不用 ccswitch）</h3>
<p>直接修改 <code>~/.claude/settings.json</code>：</p>
<pre><code class="language-json">{
  "provider": "openai",
  "model": "glm-4",
  "apiKey": "&#x3C;glm-api-key>",
  "apiUrl": "https://open.bigmodel.cn/api/paas/v4/"
}
</code></pre>
<p>各家提供商的 OpenAI 兼容格式略有不同，具体参考对方文档。一般只需改 <code>apiUrl</code> 和 <code>apiKey</code>，<code>model</code> 填对应模型名即可。</p>

<h2>目录结构</h2>
<p>Claude Code 会自动读取项目目录下的特定文件，作为项目级指令和自定义命令。</p>
<pre><code>项目根目录/
├── .claude/
│   ├── CLAUDE.md          # 项目级系统指令，每次对话自动带入上下文
│   └── commands/          # 自定义命令，每个 .md 文件就是一个命令
│       ├── review.md      # 代码审查
│       ├── refactor.md    # 重构提示
│       └── test.md        # 生成测试
├── AGENTS.md               # 项目概述、架构说明
└── ...
</code></pre>

<h2>工程化配置</h2>
<h3>全局 CLAUDE.md</h3>
<p><code>~/.claude/CLAUDE.md</code> 里放通用编码规范：</p>
<pre><code class="language-markdown"># 全局编码规范

## 通用原则

- 使用 TypeScript，严格模式
- 禁止使用 `any`
- 所有函数必须有返回类型
- 所有 API 调用必须有错误处理
- 优先使用显式返回而非隐式转换

## 文件操作

- 修改文件前先读取完整内容
- 使用精确的行号编辑，避免全重写
- 不要删除未使用的 import，除非确认无用
</code></pre>
<h3>项目级 CLAUDE.md</h3>
<p><code>.claude/CLAUDE.md</code> 里放项目特定规范：</p>
<pre><code class="language-markdown"># 项目指令

参见项目概述: [AGENTS.md](../AGENTS.md)

## 技术栈

- Next.js 15 + App Router
- Prisma ORM + PostgreSQL
- Tailwind CSS
- pnpm monorepo

## 架构约定

- API 放在 `apps/api/src`
- 组件放在 `apps/web/components`
- 工具函数放在 `packages/shared`
- 不跨应用直接引用内部路径

## 提交规范

- 使用 Conventional Commits
- 测试通过后才提交
</code></pre>

<h2>效率技巧</h2>
<h3>减少 Token 消耗</h3>
<ol>
<li><strong>使用 <code>/compact</code></strong> —— 对话变长后压缩历史，保留文件状态但清空对话记录</li>
<li><strong>小步进行</strong> —— 每次只给一个具体任务，而非整个需求文档</li>
<li><strong>限制范围</strong> —— 指定具体文件或目录，避免让 Claude 扫描整个代码库</li>
<li><strong>使用自定义命令</strong> —— 封装常用模式，减少每次重复解释</li>
</ol>
<h3>多模型切换策略</h3>
<p><strong>开发阶段</strong>：用国产模型做大部分工作，成本极低，效率很高。</p>
<p><strong>复杂逻辑</strong>：遇到国产模型反复出错的复杂算法/架构问题，切换到 Claude 或 GPT-4 处理。</p>
<p><strong>审查阶段</strong>：重要代码审查用原版 Claude，确保没有潜在问题。</p>

<h2>常见问题</h2>
<h3>Claude Code 启动失败</h3>
<pre><code class="language-bash"># 检查 Node 版本
node -v  # 需要 18+

# 重新登录
claude auth login

# 清理配置
rm -rf ~/.claude
</code></pre>
<h3>国产模型返回格式不对</h3>
<p>部分国产模型的 OpenAI 兼容层不完整，可能出现：</p>
<ul>
<li>不支持 function calling → 文件操作可能出问题</li>
<li>不支持流式输出 → 响应变慢</li>
<li>不支持某些系统提示 → 行为与原版不一致</li>
</ul>
<p>解决方案：支持较完整的模型（GLM-5.2 / Kimi K2.7 / DeepSeek V4 Pro），开发环境可以尝试更多选择。</p>
<h3>网络不稳定</h3>
<p>国产模型 API 有时会出现超时或限流：</p>
<ul>
<li>配置重试逻辑（大部分 ccswitch 已内置）</li>
<li>开发环境可以用本地模型（Ollama 等）</li>
<li>生产环境选择有 SLA 保障的提供商</li>
</ul>
<h3>文件编辑不精确</h3>
<p>Claude Code 的编辑是基于行号的精确编辑，如果文件太大或结构复杂，可能会出现偏移。建议：</p>
<ul>
<li>先让 Claude 列出要修改的行号，确认后再执行</li>
<li>对于大文件，先拆分成小任务</li>
<li>修改后用 <code>git diff</code> 检查结果</li>
</ul>
