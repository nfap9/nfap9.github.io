<h2>目录结构</h2>
<p>Claude Code 会自动读取项目目录下的特定文件，作为项目级指令和自定义命令。</p>
<pre><code>项目根目录/
├── .claude/
│   ├── CLAUDE.md          # 项目级系统指令，每次对话自动带入上下文
│   └── commands/          # 自定义命令，每个 .md 文件就是一个命令
│       ├── review.md      # 代码审查
│       ├── refactor.md    # 重构提示
│       └── test.md        # 生成测试
├── AGENTS.md               # 项目概述、架构说明
└── ...
</code></pre>

<h3>实用模式</h3>
<p><strong>仅读模式</strong></p>
<p>不想让 Claude 修改文件时：</p>
<pre><code>只读查看这个文件的结构，不要改。
</code></pre>
<p>或者用带引号的问句，不包含修改指令。</p>
<p><strong>任务式对话</strong></p>
<p>对于复杂任务，先让 Claude 列出计划，确认后逐步执行：</p>
<pre><code>实现用户登录功能，包含：1. JWT token 管理 2. 登录状态缓存 3. 登出逻辑。
请先列出实现步骤，我确认后你再开始写代码。
</code></pre>
<p><strong>结果限制</strong></p>
<p>限制输出长度，避免滚屏：</p>
<pre><code>给出最重要的 3 个修改建议，每条不超过 50 字。
</code></pre>

<h2>效率技巧</h2>
<h3>减少 Token 消耗</h3>
<ol>
<li><strong>使用 <code>/compact</code></strong> —— 对话变长后压缩历史，保留文件状态但清空对话记录</li>
<li><strong>小步进行</strong> —— 每次只给一个具体任务，而非整个需求文档</li>
<li><strong>限制范围</strong> —— 指定具体文件或目录，避免让 Claude 扫描整个代码库</li>
<li><strong>使用自定义命令</strong> —— 封装常用模式，减少每次重复解释</li>
</ol>
<h3>多模型切换策略</h3>

<p><strong>复杂逻辑</strong>：遇到国产模型反复出错的复杂算法/架构问题，切换到 Claude 或 GPT-4 处理。</p>
<p><strong>审查阶段</strong>：重要代码审查用原版 Claude，确保没有潜在问题。</p>

<h3>国产模型返回格式不对</h3>
<p>部分国产模型的 OpenAI 兼容层不完整，可能出现：</p>
<ul>
<li>不支持 function calling → 文件操作可能出问题</li>
<li>不支持流式输出 → 响应变慢</li>
<li>不支持某些系统提示 → 行为与原版不一致</li>
</ul>
<p>解决方案：支持较完整的模型（GLM-5.2 / Kimi K2.7 / DeepSeek V4 Pro），开发环境可以尝试更多选择。</p>
<h3>网络不稳定</h3>
<p>国产模型 API 有时会出现超时或限流：</p>
<ul>
<li>配置重试逻辑（大部分 ccswitch 已内置）</li>
<li>开发环境可以用本地模型（Ollama 等）</li>
<li>生产环境选择有 SLA 保障的提供商</li>
</ul>
